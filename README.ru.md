# bot-web

[English version](README.md)

Минималистичный одностраничный веб-клиент Telegram, который помогает пользователям удалить сообщения, найденные ботом [@FindMessagesBot](https://t.me/FindMessagesBot). Администраторы чатов также могут массово удалить сообщения всех участников за указанный период.

## Проверьте источник

Убедитесь, что вы получили ссылку на этот репозиторий или на веб-клиент именно напрямую с канала [@Grodno_grab](https://t.me/Grodno_grab), от аккаунта [@GrodnograbSOS](https://t.me/GrodnograbSOS) или из бота [@FindMessagesBot](https://t.me/FindMessagesBot). Если это не так, найдите в Google телеграм-канал инициативы «Суды, задержания. Гродно и область» или найдите бот поиска сообщений в крупных СМИ и проверьте, совпадает ли ссылка в точности — если нет, просим сообщить нам об этом.

## Рекомендации по использованию

1. Скопируйте целиком ссылку `https://s3.amazonaws.com/t.t/t?versionId=J1qgAjJ_PtiziFkd85OVnAjDAinW5FJJ` — она гарантирует, что версия кода останется неизменной и немодифицированной.
2. Откройте браузер в **режиме инкогнито**, вставьте и откройте ссылку.
3. Некоторые встроенные браузеры могут не работать — в таком случае рекомендуем использовать **Firefox** или **Chrome**.
4. Откройте исходный код страницы (обычно **Ctrl+U**), изучите его и убедитесь, что он совпадает в точности с тем, что находится в этом репозитории.

## Как это работает

Пользователь открывает страницу и проходит стандартную авторизацию Telegram (номер телефона → код подтверждения → при необходимости двухфакторный пароль), после чего выбирает один из двух режимов:

### Режим 1 — Удалить мои сообщения

1. Страница запускает `@FindMessagesBot` с deep-link параметром (`/start wtg`).
2. Бот отвечает зашифрованным документом (AES-CBC), содержащим список чатов и идентификаторов сообщений для удаления.
3. Пользователь подтверждает удаление; страница вызывает `deleteMessages` для каждого чата через TDLib.
4. Бот получает уведомление о результате (`/wtg_done` или `/wtg_fail`), TDLib выходит из аккаунта, а всё браузерное хранилище (localStorage, sessionStorage, IndexedDB) очищается.

### Режим 2 — Удалить сообщения как администратор чата

1. Страница загружает все чаты пользователя и находит супергруппы, в которых пользователь является администратором (или создателем) с правом удаления сообщений.
2. Пользователь выбирает чат и указывает диапазон дат (дд.мм.гггг).
3. Страница просматривает историю чата за указанный период и вызывает `deleteChatMessagesBySender` для каждого участника (кроме ботов), написавшего хотя бы одно сообщение в этом диапазоне. При этом удаляются **все** сообщения данного пользователя в чате, в том числе за пределами указанных дат.
4. Выполняется один или несколько проходов верификации, чтобы ничего не было пропущено.
5. Пользователь может выбрать другой чат или завершить работу. При завершении TDLib выходит из аккаунта, а всё браузерное хранилище очищается.

Всё выполняется **локально, прямо в браузере**. Никакие данные не отправляются на сторонние серверы.

## Стек

| Уровень | Технология |
|---------|-----------|
| Telegram MTProto | [TDLib](https://github.com/tdlib/td), скомпилированный в WebAssembly (`tdweb`) |
| Криптография | Web Crypto API — AES-CBC с ключом, производным от SHA-256 |
| Интерфейс | Чистый HTML/CSS/JS, без фреймворков |
| Хостинг | Статические файлы на Amazon S3 |

## Безопасность

- Страница отказывается работать внутри iframe или при открытии с неожиданного origin.
- Ключ шифрования полезной нагрузки формируется из `user_id` пользователя Telegram и материала из подписи к документу — документ полезен только аутентифицированному пользователю.
- Данные сессии очищаются как при штатном завершении, так и при внезапном закрытии вкладки (`pagehide` / `beforeunload`). На iOS Safari восстановление страницы из bfcache обнаруживается и приводит к перезагрузке, чтобы предотвратить устаревшую сессию.

## Сборка tdweb

Бандл `tdweb.js` и сопутствующие файлы `.wasm` / `.worker.js` собираются из исходников TDLib по официальному руководству:

> https://github.com/tdlib/td/blob/master/example/web/README.md

Инструментарий сборки (Emscripten SDK в `emsdk/` и исходники TDLib в `td/`) исключён из системы контроля версий.

## Верификация

В репозитории есть `Dockerfile.verify` и `verify.sh`, которые вместе выстраивают полную цепочку доверия за один Docker-запуск — **без ручных шагов и допущений на веру**.

```bash
git clone https://github.com/grodno-grab/bot-web && cd bot-web
docker build -f Dockerfile.verify -t tdweb-verify .
docker run --rm tdweb-verify
```

### Что проверяется (10 проверок)

**Часть 1 — Воспроизводимая сборка (4 проверки).**
Внутри чистого контейнера Ubuntu 24.04 запускается полный пайплайн сборки TDLib с нуля:
компилируется OpenSSL 1.1.0l для WebAssembly с Emscripten 3.1.1, затем TDLib
([коммит `af0cb1d3`](https://github.com/tdlib/td/commit/af0cb1d30a1e5cb1a10cd83b48998ca9ea9ce249), версия 1.8.62)
компилируется в WASM, после чего webpack собирает NPM-пакет `tdweb` — строго по [официальной инструкции TDLib](https://github.com/tdlib/td/blob/master/example/web/README.md).
SHA-256 хэши четырёх результирующих файлов (`tdweb.js`, два `.worker.js`, `.wasm`) сравниваются с копиями, закоммиченными в репозиторий.

**Часть 2 — Проверка деплоя (6 проверок).**
Скрипт берёт версионированный S3-URL из этого README, скачивает живую страницу и сравнивает с `index.html`, затем извлекает из страницы versionId каждого из пяти ассетов (`tdweb.js`, `styles.css`, два воркера, `wasm`), скачивает каждый с S3 и сравнивает с локальным файлом репозитория.

Вместе обе части доказывают: файлы, отдаваемые пользователям, побайтово совпадают с репозиторием, а файлы репозитория — немодифицированный результат стандартной сборки TDLib.
